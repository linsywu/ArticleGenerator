# ArticleGenerator — 领域词汇表

> 内容原则：一处声明，多处引用。术语在"核心领域概念"中定义，各服务节引用核心概念并补充服务专属语境。

## 核心领域概念

以下概念跨多个服务使用，在此统一定义。

### 账号（Account）
一位创作者的写作身份。一个账号 = 一个风格（按平台拆分，如"陆拾一公众号"和"陆拾一小红书"为两个独立账号）。每个账号拥有一组参考文章和一个结构化的风格画像。

### 风格画像（Style Profile）
从参考文章中蒸馏出的 7 维度结构化写作特征描述，替代自由文本。维度：
- **思维特征** — 论证习惯（归纳/演绎/类比）、常用逻辑框架
- **结构模式** — 开头偏好、段落组织方式、结尾风格
- **句式特征** — 句子长度分布、常用修辞、标点偏好、段落节奏
- **词汇偏好** — 高频词、禁用词、语气词使用频率
- **论据类型** — 偏好用个人经历/数据/案例/引用论证
- **禁忌清单** — 该作者绝不会写的内容
- **留白程度** — 道理讲几分、是否留悬念/不解释

### 蒸馏（Distill）
从参考文章自动提取写作特征、生成风格画像的过程。支持结构化输出和动态进化。涉及服务：后端 API（触发）→ LLM 服务（执行）。

### 文章（Article）
由系统生成的完整文章。经历 生成 → 评审 → 微调 循环后达到质量标准。涉及服务：后端 API（存储/编排）→ LLM 服务（生成/微调）→ 管理后台（展示/操作）。

### 评审（Review）
对比式评分：将生成文章与账号参考文章对比，分维度打分（逻辑性/风格匹配/可读性/原创性），输出修改建议。

### 微调（Refine）
根据评审结果自动调整文章。系统自动生成修改指令，用户可选择介入加额外指令。

### 评审-微调循环（Review-Refine Loop）
评审 → 展示评分和修改建议 → 用户确认 → 微调 → 再评审，直到评分达到阈值。每轮都停，用户掌控每一步。

### 通过阈值（Pass Threshold）
文章评审评分达到预设分数（如 ≥85 分）时自动标记为通过，退出循环。

### 灵感墙（Inspiration Wall）
降级后的热点浏览功能。不自动抓取，用户偶尔浏览热榜获取灵感，可一键带入生成流程。涉及服务：后端 API（数据提供）→ 管理后台（展示）。数据来源：热点抓取服务。

### 发布（Publish）
终稿归档 + 手动复制。系统提供文本复制和发布状态标记，不自动发布到第三方平台。

### 动态风格进化（Dynamic Style Evolution）
风格画像随已通过文章持续优化。两种触发方式：
- **自动建议** — 累积 5-10 篇通过文章后提示画像更新建议，用户确认
- **手动触发** — 用户随时重新蒸馏
涉及服务：后端 API（检测触发）→ LLM 服务（执行蒸馏）。

## 后端 API 概念（ArticleGeneratorService）

以下概念由后端 API 服务独有或主要在此服务中操作。核心领域概念见上文。

### 想法（Idea）
用户手动输入的创作起点。两种输入形式：
- **URL 输入** — 粘贴链接，系统提取核心论点 + 找到可反驳/延伸的点
- **手动描述** — 用一两句话描述想写的主题

### 写作方向（Writing Direction）
同一主题的不同切入角度。一句话级粒度。如"从个人学编程经历切入，讲工具迭代的必然性"。

### 大纲（Outline）
文章的极简骨架：5-8 个要点，每个要点一句话概括。用户确认后生成全文。

## 工作流

```
选账号 → 输入想法 → 生成写作方向 → 选方向 → 生成大纲 → 确认大纲 → 生成全文 → [评审 → 确认 → 微调] ↻ 直到通过 → 终稿 → 发布
```

## 内容采集概念

### 赛道（Track）
内容分类的顶层体系。一级赛道（如"AI科技"）下包含多个二级赛道（如"大模型"、"AI应用"）。每个赛道可配置热门关键词和禁用关键词，用于后续内容筛选。

### 公众号（MP Account）
采集目标——对标竞品的微信公众号。每个公众号关联一个或多个赛道。有两种接入方式：公众号名称搜索自动获取信息、文章链接解析。关键采集参数：fakeid（公众号唯一标识）、biz。

### 采集凭证（Collection Credential）
微信公众号后台的登录态凭证（token + cookie）。由用户手动填入，系统定时检测有效性。状态流转：normal → expiring_soon → expired。

### 采集任务（Collection Task）
一次采集执行的配置：目标范围（赛道+公众号多级选择）、采集模式（历史同步/增量同步）、执行方式（手动/定时/间隔）。

### 素材（Material）
从外部公众号采集的参考文章。区别于 AI 生成的「文章（Article）」——素材是对标参考，文章是原创产出。素材存储原始 HTML，使用时按需解析为 Markdown。

### 采集（Collection）
从目标公众号拉取历史文章的过程。采集仅保存原始 HTML，不做图片下载或内容分析。执行策略：低频（每天一次）、防风控（15s+间隔+抖动）、去重（URL+内容哈希）。

## 非目标（明确不做）

- 自动热点发现与热搜抓取（灵感墙已从自动降级为按需浏览）
- 自动发布到第三方平台（公众号/小红书等）
- 全自动无人干预的内容生产
