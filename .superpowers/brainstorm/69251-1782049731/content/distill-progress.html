风格蒸馏 — 进度反馈方式

当前：点击"蒸馏风格" → Toast "已提交，请稍后刷新" → 用户盲等。哪种反馈方式更合理？

A

轮询 + 进度条

前端定时轮询任务状态（pending → running → completed/failed），UI 显示进度条 + 当前阶段文字（如"正在分析文章内容..."）。简单可靠，无需额外基础设施。

B

SSE 实时推送

后端通过 Server-Sent Events 推送蒸馏进度（每完成一个维度推送一次），前端实时更新进度条和各维度状态。体验最好但需要后端支持 SSE。

C

状态标签 + 自动刷新

蒸馏按钮变成状态标签（"蒸馏中..."带 loading 动画），自动轮询直到完成，完成后自动展示画像结果。不显示进度条，只显示进行中/完成/失败。

D

任务列表 + 通知

页面顶部或侧边加一个"任务中心"图标，蒸馏任务提交后进入任务列表，显示状态和时间。完成后通知角标提醒用户查看。
