📋 模块七：文章创作增强（修订版）

基于对现有 CreateView 5步向导流程的充分理解

现有流程回顾（已经做得很好）

Step 1
选择账号
→
Step 2
输入想法
→
Step 3
写作方向 ✅论点风暴
→
Step 4
确认大纲 ✅可编辑
→
Step 5
生成全文 ✅含轮询

真正需要增强的 4 个点

1. Step 2 增强：AI 参数面板

在「输入想法」步骤增加可折叠的「高级设置」：

  • 字数 — 读取账号的 word_count 字段作为默认值，可手动覆盖（如：1500字左右、2000-3000字、3000字以上）
  • 写作形式 — 深度分析 / 观点评论 / 实用教程 / 故事叙事
  • 原创度 — 保守←→激进创新的 slider
  • （风格模板已有：Step 1 选择账号 = 选择风格，不需要额外参数）
2. 生成输出增强：标题候选 + 摘要 + SEO

现有 /generate/titles 已实现但未集成到 UI 流程中。生成全文后应展示：

  • 3 个候选标题 — 调用已有的 /generate/titles，展示在文章上方可选
  • 摘要 — 200字文章摘要（新增 LLM 场景）
  • SEO 关键词 — 5-10个标签（新增 LLM 场景）
  • （配图建议：用户反馈三方文章图片参考性低，同理 AI 生成的文章配图建议价值有限。可做可不做。）
3. 版本管理

现有微调(refine)是覆盖式重写。改为版本累积：

  • 新增 article_versions 表（article_id + version_number + content + params 快照）
  • 每次 refine 不是覆盖 articles.content，而是新增一条 version 记录
  • articles.content 保持为 current version，评审/发布用 current
  • ArticleEditorDialog 增加版本切换下拉框
4. 外部入口集成

让创作中心能接收外部传入的上下文：

  • 素材中心论点风暴 → 一键带入创作：路由携带预填参数（标题、目标人群、核心观点、大纲），直接跳转到 CreateView Step 3 或 Step 4
  • 知识库侧边栏：Step 4（确认大纲）和 Step 5（生成全文）时，侧边栏可检索观点/案例/金句/标题，用于参考引用
📊 增强工作量评估：
  • AI 参数面板：~1天（前端为主，后端 /trigger 增加参数透传）
  • 标题/摘要/SEO 输出：~1天（前端 Step 5 展示 + 后端新增 summary/seo 场景）
  • 版本管理：~2天（新表 + 后端 API + 前端版本切换 UI）
  • 外部入口集成：~1天（路由参数 + 侧边栏组件）
